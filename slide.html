📜 History of Git

  • Creator: Linus Torvalds (The mastermind behind the Linux Operating System).
  • Year: Created in 2005 (Astonishingly built in just about two weeks!).
  • Why it was created: To track changes for the Linux kernel project after their access to the previous tool (BitKeeper) was revoked.
  • Main Goal: To establish a blazing fast, decentralized architecture that works efficiently offline.

☁️ GitHub

  • Creators: Founded by Tom Preston-Werner, Chris Wanstrath, PJ Hyett, and Scott Chacon.
  • Year: Officially launched to the public in 2008.
  • Why it was created: Git was strictly a command-line interface tool. GitHub was created to bring it to the web with an easy UI for seamless collaboration.
  • Current Owner: Acquired by Microsoft in 2018 for a milestone value of $7.5 billion.

⚖️ Git vs GitHub

  • Git (2005): The core version control engine/software you download onto your system. It logs all adjustments offline locally.
  • GitHub (2008): An online cloud-hosted platform built to manage your Git projects. It connects developers across the globe.

1️⃣ Repository (Repo)

A Repository is the centralized digital directory or folder that stores your entire workspace, containing:

  • All your source code files, documents, and project configurations.
  • The complete history logs tracking every incremental revision.
  • Project summaries and documentation files (such as a README.md file).

Quick Steps to Create: Click the + top-right icon on GitHub → Choose "New repository" → Set accessibility configurations (Public/Private) → Click Create.

2️⃣ Git Clone

Git Clone handles the initial operational setup for a developer on their local computer.

  • Definition: Downloading an existing remote online project directory from GitHub onto your computer for the first time.
  • The Mechanism: Running the command downloads all target asset files along with the historical timeline records.
  • Command Usage: git clone <repository-url>
  • Key Difference: Clone acts as the foundational building setup, while subsequent updates are fetched using a simple pull.

💻 Terminal

A commit records a precise snapshot or historical save-state of your active project structure.

  • git add . : Stages and packs all updated project files to prepare for storage.
  • git add home.html : Explicitly target packages only a specific single file.
  • git commit -m "Message" : Executes the save snapshot using a clear note summarizing the modification.

Common terminal commands:

git init
git remote add origin https://github.com/muthuselvam-1125/presentation.html.git
git add .
git commit -m "first commit"
git branch -M main
git push -u origin main

Core Value: It functions like a timeline checkpoint. If a catastrophic error occurs, you can jump back to any previous working commit seamlessly.

4️⃣ Collaboration (Team Work)

Collaboration describes the multi-developer approach where a unified remote workspace handles concurrent pushes.

  • Project owners explicitly invite team members via "Settings" → "Collaborators".
  • The Production Flow:
    • Developer A pushes their local code changes up to the cloud repo.
    • Developer B pulls the latest baseline code down onto their computer.
    • Developer B builds additional features on top of it and pushes the work back up.

5️⃣ Branching

Branching provides a private, isolated sandboxed workspace to build features without disrupting functional code.

  • The Concept: Creates a parallel development universe branching away from your production-ready main branch.
  • Safety: Code changes, experiments, or errors inside a feature branch will never impact or break the stable baseline code.
  • Merging: Once the brand new updates are tested and verified, the changes are securely integrated back into the core version.

6️⃣ Push & Pull

  • Push (git push): Uploads your local milestones and code commits from your computer up to the GitHub repository online.
  • Pull (git pull): Downloads and instantly applies incoming edits from the online repo onto your local computer. It is critical for staying synced.
